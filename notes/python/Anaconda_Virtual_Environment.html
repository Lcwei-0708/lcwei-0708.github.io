Anaconda虛擬環境簡介

在使用Python進行開發時，很常會因為每個專案所需的Python版本或是套件版本不同，導致在同一個環境下造成衝突，又或是每個專案所使用到的套件不同，但是如果將所有專案的套件都放在同一個環境下，很容易造成管理不便或是匯出專案時過於肥大，因此Anaconda提供建置虛擬環境來進行開發的功能，可以依據每個專案的需求，來各自安裝需要的Python版本與套件。

管理虛擬環境的相關操作

創建環境
Shell 
conda create -n myenv python=3.9

conda create -n <環境名稱> python=<Python版本>

輸入完指令後會詢問是否同意安裝以下這些新套件，回答：y

到這邊就表示已經建置好新的虛擬環境了

查看目前有哪些虛擬環境
Shell 
conda env list

所有虛擬環境列表，星號代表目前所屬環境

啟動虛擬環境
Shell 
conda activate myenv

conda activate <環境名稱>

從原本的base環境切換到myenv環境

離開虛擬環境
Shell 
conda deactivate

從myenv環境回到base環境中

刪除整個虛擬環境
Shell 
conda env remove -n myenv

conda env remove -n <環境名稱>

虛擬環境已刪除

其他常用Conda指令

檢查Conda版本
Shell 
conda -V

查看Conda完整資訊
Shell 
conda info

更新Conda版本
Shell 
conda update conda

安裝套件
Shell 
conda install numpy

conda install <套件名稱>

更新套件
Shell 
conda update numpy

conda update <套件名稱>

刪除套件
Shell 
conda uninstall numpy

conda uninstall <套件名稱>

查看虛擬環境中的套件
Shell 
conda list

匯入虛擬環境
Shell 
conda env create -f environment.yml

conda env create -f <環境檔案名稱>

匯出虛擬環境
Shell 
conda env export > environment.yml

conda env export > <環境檔案名稱>
