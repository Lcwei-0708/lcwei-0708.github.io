移動到專案目錄下

大部分的操作指令都需在專案目錄下輸入。

cd D:/project

建立本地的數據庫

git init

設定遠端數據庫

連接遠端數據庫
  • 複製既有的遠端數據庫
    git clone git@github.com:Lcwei-0708/project.git

  • 新增一個遠端數據庫
    git remote add origin git@github.com:Lcwei-0708/project.git

其他遠端設定指令
git remote

git remote -v

git remote show <遠端數據庫名稱>

git remote update

git remote set-url <遠端數據庫名稱> <新的遠端數據庫URL>

git remote rename <舊的遠端數據庫名稱> <新的遠端數據庫名稱>

git remote remove <遠端數據庫名稱>

分支管理的相關指令

查看分支
git branch

建立分支
git branch branch1

刪除分支
git branch -d branch1

回復被刪除的分支

git checkout -b branch2 a7066f5

更改分支名稱
git branch -m branch1 branch2

切換分支
git checkout branch2

切換到上一個分支
git checkout -

建立並且切換到新的分支
git checkout -b branch2

使用Merge合併分支

將其他分支合併到目前分支
git merge branch2

強制Git創建一個新的合併提交
git merge --no-ff branch2

繼續執行合併動作
git merge --continue

恢復到合併前的狀態
git merge --abort

使用Rebase合併分支

將目前分支提交到其他分支
git rebase branch2

編輯、刪除、合併提交訊息
git rebase -i branch2

繼續執行合併動作
git rebase --continue

恢復到合併前的狀態
git rebase --abort

管理專案版本的常用指令

將文件加入索引中

git add .

查看索引的狀態
git status

將索引內的檔案提交到本地數據庫
git commit -m "2023/06/26"

查看提交的歷史訊息
git log

將檔案推送到遠端數據庫
git push origin master

從遠端數據庫拉取檔案
git pull origin master
