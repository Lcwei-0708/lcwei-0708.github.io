Web Driver下載

因為Selenium是透過Web Driver進行爬蟲，所以必須先下載好Web Driver，以下為不同瀏覽器所提供的Web Driver：

瀏覽器 下載網址
Chrome https://sites.google.com/chromium.org/driver/
Edge https://developer.microsoft.com/en-us/microsoft-edge/tools/webdriver/
Firefox https://github.com/mozilla/geckodriver/releases
Safari https://webkit.org/blog/6900/webdriver-support-in-safari-10/

設定Web Driver

Python 
from selenium import webdriver

Import Web Driver套件

Python 
options = webdriver.ChromeOptions()

創建一個設定Chrome瀏覽器的物件

Header設定
Python 
headers = {
	'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/91.0.4472.124 Safari/537.36',
	'Accept-Language': 'en-US,en;q=0.9',
	'Referer': 'https://example.com'
}

option.add_argument(headers)

隱藏瀏覽器視窗
Python 
options.add_argument("--headless")

禁用GPU加速
Python 
options.add_argument("--disable-gpu")

Expected Conditions（EC）

因為有些網頁是透過Ajax方式回傳資料，所以第一時間是抓取不到元素的，必須等待Ajax資料讀取完畢才能抓取元素，所以可以透過此方法來等待元素完全載入。

Python 
from selenium.webdriver.support import expected_conditions as EC

Import Expected Conditions套件

    常用的等待條件：
  • title_is()：等待網頁標題完全符合指定的title。
  • title_contains()：等待網頁標題包含指定的substring。
  • presence_of_element_located()：等待元素定位器出現在網頁中。
  • visibility_of_element_located()：等待元素定位器在網頁中可見。
  • visibility_of()：等待元素在網頁中可見。
  • presence_of_all_elements_located()：等待所有符合定位器的元素出現在網頁中。
  • text_to_be_present_in_element()：等待指定元素中出現指定的文本。
  • text_to_be_present_in_element_value()：等待指定元素的value屬性中出現指定的文本。
  • frame_to_be_available_and_switch_to_it()：等待iframe元素可用並切換到該iframe。
  • invisibility_of_element_located()：等待元素定位器在網頁中不可見。
  • element_to_be_clickable()：等待元素可被點擊。
  • staleness_of()：等待元素變得過時（不再出現在DOM中）。
  • element_to_be_selected()：等待元素被選擇。
  • element_located_to_be_selected()：等待元素定位器被選擇。
  • element_selection_state_to_be()：等待元素的選擇狀態符合指定的狀態。
  • element_located_selection_state_to_be()：等待元素定位器的選擇狀態符合指定的狀態。
  • alert_is_present()：等待彈出窗口出現。

Explicit Waits與Implicit Waits

不管是使用Explicit Waits還是Implicit Waits，如果超過了指定的時間仍未找到元素，系統則會拋出TimeoutException異常。

Explicit Waits

Explicit Waits可以針對特定的元素或條件進行等待，可以更精確地控制等待的時間和條件，搭配Expected Conditions來指定等待條件，當條件滿足時才繼續執行後續的操作。

Python 
from selenium.webdriver.support.ui import WebDriverWait

Import Wait 套件

Python 
wait = WebDriverWait(driver, 10)
element = wait.until(EC.presence_of_element_located((By.TAG_NAME, "h1")))

Implicit Waits

Implicit Waits是一種全局性的等待方式，對之後的所有元素定位操作都生效，一樣給予一定的等待時間，如果在指定的時間內找到了元素，則立即執行相應的操作

Python 
driver.implicitly_wait(10)
element = driver.find_element(By.TAG_NAME, "h1")

執行JavaScript操作

在Selenium中提供了execute_script()Function來執行JavaScript的程式碼，讓使用者可以進行一些更進階的操作。

捲動視窗到最底部
Python 
driver.execute_script("window.scrollTo(0, document.body.scrollHeight);")

將scrollTo(X, Y)的Y座標設定為整個網頁內容的垂直滾動高度

執行自定義Function
Python 
driver.execute_script("function myFunction() { console.log('Hello, world!'); } myFunction();")

定義一個新的Function，並且呼叫它

修改元素屬性
Python 
driver.execute_script("document.getElementById('elementId').setAttribute('attributeName', 'attributeValue');")

修改指定元素的屬性

Web Driver相關API

Action Chains

Action Chains是Selenium提供的一種操作方式，用於執行一系列的鍵盤和滑鼠動作，例如：鼠標移動、鍵盤按鍵、滾動等。

Python 
from selenium.webdriver.common.action_chains import ActionChains

Import Action Chains套件

Python 
actions = ActionChains(driver)

創建一個Action Chains物件

    Action Chains的常見操作：
  • click(on_element=None)：點擊元素，若on_element為None，則點擊當前滑鼠位置。
  • click_and_hold(on_element=None)：按住元素的左鍵。
  • context_click(on_element=None)：在元素上執行右鍵點擊。
  • double_click(on_element=None)：雙擊元素。
  • drag_and_drop(source, target)：在source元素上按住左鍵，然後移動到target元素並釋放左鍵。
  • drag_and_drop_by_offset(source, xoffset, yoffset)：在source元素上按住左鍵，然後按指定的偏移量移動，並釋放左鍵。
  • key_down(value, element=None)：發送按鍵按下事件，僅用於修飾鍵（如 Control、Alt 和 Shift）。
  • key_up(value, element=None)：釋放按鍵。
  • move_by_offset(xoffset, yoffset)：移動滑鼠到相對於當前滑鼠位置的偏移量。
  • move_to_element(to_element)：移動滑鼠到元素的中心位置。
  • move_to_element_with_offset(to_element, xoffset, yoffset)：移動滑鼠到指定元素的偏移位置。
  • pause(seconds)：暫停指定秒數的所有動作。
  • perform()：執行所有已存儲的動作。
  • release(on_element=None)：在元素上釋放已按住的滑鼠按鍵。
  • reset_actions()：清除已經在本地和遠程端存儲的動作。
  • scroll(x, y, delta_x, delta_y, duration=0, origin='viewport')：將滾輪滾動訊息發送到瀏覽器進行處理。
  • scroll_by_amount(delta_x, delta_y)：按指定的值進行滾動，以可視區域的左上角為原點。
  • scroll_from_origin(scroll_origin, delta_x, delta_y)：基於指定的原點進行滾動，原點可以是元素的中心或可視區域的左上角加上任何偏移量。
  • scroll_to_element(element)：如果元素在可視區域之外，將元素的底部滾動到可視區域的底部。
  • send_keys(*keys_to_send)：將鍵盤按鍵發送給當前聚焦的元素。
  • send_keys_to_element(element, *keys_to_send)：將鍵盤按鍵發送給指定元素。

Proxy

Proxy包含了三個類別，分別是：Proxy、ProxyType和ProxyTypeFactory。

Proxy：包括了代理類型和必要的代理設定。

Python 
from selenium.webdriver.common.proxy import Proxy

Import Proxy套件

Python 
proxy = Proxy()

創建一個Proxy物件

    Proxy的常見操作：
  • add_to_capabilities(capabilities)：將代理設定添加到指定的能力（capabilities）中。
  • auto_detect()：返回自動偵測。
  • ftp_proxy()：返回FTP代理。
  • http_proxy()：返回HTTP代理。
  • no_proxy()：返回無代理。
  • proxy_autoconfig_url()：返回代理自動配置URL。
  • proxy_type()：返回代理類型，以ProxyType表示。
  • socks_password()：返回SOCKS代理的密碼。
  • socks_proxy()：返回SOCKS代理。
  • socks_username()：返回SOCKS代理的使用者名稱。
  • socks_version()：返回SOCKS代理的版本。
  • ssl_proxy()：返回HTTPS代理。

ProxyType：定義了可能的代理類型，並提供了相應的屬性。

Python 
from selenium.webdriver.common.proxy import ProxyType

Import ProxyType套件

    ProxyType的常見操作：
  • ProxyType.load(value)：靜態方法，根據給定的值載入對應的ProxyType物件。
  • ProxyType.DIRECT：直接連接。
  • ProxyType.MANUAL：手動設定代理。
  • ProxyType.PAC：PAC（Proxy Auto-Config）代理。
  • ProxyType.RESERVED_1：保留。
  • ProxyType.AUTODETECT：自動偵測代理。
  • ProxyType.SYSTEM：系統代理。
  • ProxyType.UNSPECIFIED：未指定代理類型。
ProxyType的ff_value與string值對照表：

PAC 2 'PAC'
RESERVED_1 3 'RESERVED_1'

Exceptions

Selenium提供了許多Web Driver發生異常的種類，可以用來針對Web Driver的操作進行異常捕捉。

Python 
from selenium.common.exceptions import <例外名稱>

Import Selenium Exceptions套件

    Selenium提供的異常種類列表：
  • ElementClickInterceptedException：當要點擊的元素被其他元素遮擋而無法完成點擊操作時，會拋出此異常。
  • ElementNotInteractableException：當一個元素存在於DOM中，但由於繪製順序的原因，與該元素互動時會觸發其他元素，導致無法與目標元素互動時，會拋出此異常。
  • ElementNotSelectableException：當一個元素存在於DOM中，但由於不可見，無法與該元素進行互動時，會拋出此異常，例如點擊或讀取不可見元素的文本時。
  • ImeActivationFailedException：當啟動輸入法引擎（Input Method Engine）失敗時，會拋出此異常。
  • ImeNotAvailableException：當輸入法引擎（Input Method Engine）不可使用時，會拋出此異常。
  • InsecureCertificateException：當導航引發用戶端遇到憑證警告時，通常是由於過期或無效的TLS憑證導致的，會拋出此異常。
  • InvalidArgumentException：當傳遞給命令的參數無效或格式錯誤時，會拋出此異常。
  • InvalidCookieDomainException：當嘗試添加一個與當前 URL 不同域的 cookie 時，會拋出此異常。
  • InvalidCoordinatesException：當提供給互動操作的坐標無效時，會拋出此異常。
  • InvalidElementStateException：當一個元素處於無效狀態時，導致無法完成命令時，會拋出此異常。
  • InvalidSelectorException：當用於查找元素的選擇器未返回Web Element時，會拋出此異常，通常發生在選擇器為XPath表達式且該表達式無效或不選擇 Web Element 時。
  • InvalidSessionIdException：當給定的Session ID不在活動Session列表中時，表示該Session不存在或不活動時，會拋出此異常。
  • InvalidSwitchToTargetException：當要切換的Frame或Window不存在時，會拋出此異常。
  • JavascriptException：當執行用戶提供的JavaScript過程中發生錯誤時，會拋出此異常。
  • MoveTargetOutOfBoundsException：當Actions Chains的move()方法的目標無效，即超出了文檔範圍時，會拋出此異常。
  • NoAlertPresentException：當切換到沒有提示的Alert時，會拋出此異常。
  • NoSuchAttributeException：當找不到元素的屬性時，會拋出此異常。
  • NoSuchCookieException：當在當前瀏覽上下文的活動文檔的相關Cookie中找不到與給定的Path名稱匹配的Cookie時，會拋出此異常。
  • NoSuchElementException：當找不到元素時，會拋出此異常。
  • NoSuchFrameException：當要切換的Frame不存在時，會拋出此異常。
  • NoSuchShadowRootException：當嘗試訪問元素的Shadow Root但該元素沒有附加Shadow Root時，會拋出此異常。
  • NoSuchWindowException：當要切換的視窗不存在時，會拋出此異常。
  • ScreenshotException：當無法進行螢幕截圖時，會拋出此異常。
  • SeleniumManagerException：當與Selenium管理器互動出現問題時，會拋出此異常。
  • SessionNotCreatedException：當無法創建新的Session時，會拋出此異常。
  • StaleElementReferenceException：當對一個已經Stale的元素的引用時，會拋出此異常，Stale指的是該元素不再出現在頁面的DOM中。
  • TimeoutException：當命令在足夠的時間內未完成時，會拋出此異常。
  • UnableToSetCookieException：驅動程序無法設置Cookie時，會拋出此異常。
  • UnexpectedAlertPresentException：當出現意外的提示框時，會拋出此異常，通常是由於意外的模態導致網頁驅動程序無法執行命令時。
  • UnexpectedTagNameException：當一個支持類未獲取到預期的Web元素時，會拋出此異常。
  • UnknownMethodException：當請求的命令與已知的URL匹配，但未匹配到該URL的任何方法時，會拋出此異常。
  • WebDriverException：當Web Driver發生時，會拋出此異常，這個異常是所有Exception的父類別。
