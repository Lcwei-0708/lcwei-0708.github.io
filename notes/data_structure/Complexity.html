什麼是複雜度？

複雜度指的是一個演算法執行所需的資源量，通常分為時間複雜度和空間複雜度兩種，複雜度分析在選擇和設計演算法時起著至關重要的作用。

  • 效能評估：通過複雜度分析，我們可以評估不同演算法的效能。這使得我們能夠選擇最合適的演算法來處理特定的問題。
  • 優化和改進：複雜度分析可以幫助我們識別並改進效能較差的部分，以使得演算法在處理大型資料或實時系統中更為高效。
  • 預測行為：複雜度分析也可以用於預測演算法在不同輸入大小下的行為，這對於規劃系統資源和預測執行時間很有幫助。

什麼是Big O？

Big O就像一種簡單的衡量標準，讓我們能夠估計一個演算法的效率有多高。它告訴我們的是，當我們的輸入資料變大時，這個演算法是否會變得更慢。

    常見的Big O表示：
  • O(1) - 常數時間：代表無論處理多少資料，執行時間都差不多。
  • O(n) - 線性時間：代表處理資料的時間與資料量成正比。
  • O(n^2) - 平方時間：代表當資料量變大時，執行時間會以平方的速度增長。
  • O(log n) - 對數時間：當資料量變大時，執行時間以對數方式增長，這表示隨著資料量增加，所需的操作次數變得越來越少。
  • O(n log n) - 線性對數時間：複雜度較高但仍然很有效率的演算法，它比平方時間更好，但比線性時間差一些。
  • O(2^n) - 指數時間複雜度：隨著輸入數據的增加，執行時間將以指數方式增長，這種複雜度通常出現在涉及到遞迴和分支的情況，其中每一個遞迴調用會產生指數級別的子問題。
  • O(n!) - 階乘時間複雜度：隨著輸入數據的增加，執行時間將以階乘方式增長，這是一種極端的情況，通常出現在需要進行所有可能排列或組合的情況下。

時間複雜度

時間複雜度（Time Complexity）是指一個演算法在執行時所需的時間量，通常以輸入大小（n）的函數形式表示。它衡量了隨著輸入大小增加，演算法執行時間的增長速度。

    常見的時間複雜度：
  • O(1) - 常數時間
    Python 
    def constant_time_example():
	print('O(1) - Constant Time')

    這個函數執行的時間與輸入的大小無關，無論輸入是多大，這個函數的執行時間都保持不變。

  • O(n) - 線性時間
    Python 
    def linear_time_example(n):
	for i in range(n):
		print('O(n) - Linear Time')

    這個函數的執行時間會隨著輸入n的增加而線性增長，當n增加一單位，執行時間也會增加一單位

  • O(n^2) - 平方時間
    Python 
    def quadratic_time_example(n):
	for i in range(n):
		for j in range(n):
			print('O(n^2) - Quadratic Time')

    這個函數的執行時間會隨著輸入n的增加而成平方增長，當n增加一單位，執行時間會增加n個單位

  • O(log n) - 對數時間
    Python 
    def logarithmic_time_example(n):
	while n > 1:
		n = n // 2
		print('O(log n) - Logarithmic Time')

    這個函數的執行時間會隨著輸入n的對數增長，每次迭代，n的值會變為一半，因此執行時間以對數方式增長

  • O(n log n) - 線性對數時間
    Python 
    def linear_logarithmic_time_example(n):
	for i in range(n):
		j = 1
		while j < n:
			j = j * 2
			print('O(n log n) - Linear Logarithmic Time')

    這個函數的執行時間會隨著輸入n的增加以及以對數方式增長，外部循環使得執行時間線性增長，內部循環使得執行時間以對數方式增長

  • O(2^n) - 指數時間
    Python 
    def exponential_time_example(n):
	def exponential(n):
		if n == 0:
			return 1
		else:
			return 2 * exponential(n-1)
	exponential(n)
	print('O(2^n) - Exponential Time')

    這個函數的執行時間會隨著輸入n的指數增長。這是一個非常高的時間複雜度，執行時間隨著n的增加急劇增長

  • O(n!) - 階乘時間
    Python 
    def factorial_time_example(n):
	result = 1
	for i in range(1, n+1):
		result *= i
	print('O(n!) - Factorial Time')

    這個函數計算n的階乘，因此執行時間隨著n的增加以階乘方式增長

空間複雜度

空間複雜度（Space Complexity）是指一個演算法在執行過程中所需要的記憶體空間量，與時間複雜度一樣，我們關心的是演算法在處理不同規模的輸入時，記憶體使用量的變化。

    常見的時間複雜度：
  • O(1) - 常數空間
    Python 
    def constant_space_example():
	a = 10
	b = 20
	c = a + b

    不論輸入大小如何變化，這個Function所需的記憶體空間保持不變

  • O(n) - 線性空間
    Python 
    def linear_space_example(n):
	numbers = []
	for i in range(n):
		numbers.append(i)

    這個Function會創建一個包含n個元素的列表，所需的記憶體空間與n成正比

  • O(n^2) - 平方空間
    Python 
    def quadratic_space_example(n):
	matrix = [[0 for _ in range(n)] for _ in range(n)]

    這個Function會創建一個n*n的矩陣，所需的記憶體空間隨著n的平方成長

  • O(log n) - 對數空間
    Python 
    def logarithmic_space_example(n):
	if n <= 1:
		return n
	else:
		return logarithmic_space_example(n // 2)

    這個遞迴Function每次調用都會在堆疊中佔用一些空間，但空間隨著遞迴深度增加而對數增長

  • O(n log n) - 線性對數空間
    Python 
    def linear_logarithmic_space_example(n):
	for i in range(n):
		for j in range(n // 2):
			print(i, j)

    這個Function的記憶體使用量與n乘以log n成正比

  • O(2^n) - 指數空間
    Python 
    def exponential_space_example(n):
	if n <= 0:
		return 1
	else:
		return exponential_space_example(n - 1) + exponential_space_example(n - 1)

    這個遞迴Function每次調用都會在堆疊中佔用大量的空間，隨著n的增加，記憶體使用量急劇增加

  • O(n!) - 階乘空間
    Python 
    def factorial_space_example(n):
	if n <= 1:
		return 1
	else:
		return n * factorial_space_example(n - 1)

    這個遞迴函數每次調用都會在堆疊中佔用大量的空間，隨著n的增加，記憶體使用量以階乘方式增長
