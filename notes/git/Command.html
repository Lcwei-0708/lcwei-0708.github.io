移動到專案目錄下

大部分的操作指令都需在專案目錄下輸入，所以要先移動到專案的目錄下。

Shell 
cd D:/project

cd <專案路徑>

目錄已經切換到專案底下

建立本地的數據庫

Shell 
git init

專案的資料夾內多出了.git的資料夾（預設是隱藏的）

設定遠端數據庫

複製專案的SSH URL

連接遠端數據庫
  • 複製既有的遠端數據庫
    Shell 
    git clone git@github.com:Lcwei-0708/project.git

    git clone <遠端數據庫URL>

  • 新增一個遠端數據庫
    Shell 
    git remote add origin git@github.com:Lcwei-0708/project.git

    git remote add <遠端數據庫名稱> <遠端數據庫URL>

其他遠端設定指令
Shell 
git remote

查看已設定的遠端數據庫

Shell 
git remote -v

查看已設定的遠端數據庫URL

Shell 
git remote show <遠端數據庫名稱>

查看指定遠端數據庫的詳細資訊

Shell 
git remote update

獲取遠端數據庫最新的分支與提交

Shell 
git remote set-url <遠端數據庫名稱> <新的遠端數據庫URL>

更改遠端數據庫URL

Shell 
git remote rename <舊的遠端數據庫名稱> <新的遠端數據庫名稱>

重新命名遠端數據庫

Shell 
git remote remove <遠端數據庫名稱>

刪除遠端數據庫

分支管理的相關指令

查看分支
Shell 
git branch

建立分支
Shell 
git branch branch1

git branch <分支名稱>

刪除分支
  • 若要刪除的分支本身有commit，且未被合併（merge），則無法使用-d來進行刪除，必須使用-D才能刪除。
Shell 
git branch -d branch1

git branch -d <分支名稱>

回復被刪除的分支

刪除分支時Git會給予一個ID，如果要回復被刪除的分支的話，只要重新建立分支，並且在分支名稱後面加上被刪除時的ID，這樣這個分支之前的紀錄就都會復原了。

查詢被刪除的分支ID

Shell 
git checkout -b branch2 a7066f5

git branch -b <分支名稱> <被刪除時的ID>

更改分支名稱
Shell 
git branch -m branch1 branch2

git branch -m <舊的分支名稱> <新的分支名稱>

切換分支
Shell 
git checkout branch2

git checkout <分支名稱>

切換到上一個分支
Shell 
git checkout -

git checkout <分支名稱>

建立並且切換到新的分支
Shell 
git checkout -b branch2

git checkout -b <分支名稱>

使用Merge合併分支

將其他分支合併到目前分支
  • 執行此指令會將指定的分支（branch-name）合併到當前所在的分支，所以必須先切換到要合併的分支上。
Shell 
git merge branch2

git merge <要被合併的分支名稱>

強制Git創建一個新的合併提交
Shell 
git merge --no-ff branch2

git merge --no-ff <分支名稱>

繼續執行合併動作
Shell 
git merge --continue

在解決合併衝突後，使用此指令繼續執行合併操作

恢復到合併前的狀態
Shell 
git merge --abort

如果在合併過程中遇到問題或錯誤，可以使用此指令放棄合併操作

使用Rebase合併分支

將目前分支提交到其他分支
Shell 
git rebase branch2

git rebase <要提交到的分支名稱>

編輯、刪除、合併提交訊息
Shell 
git rebase -i branch2

git rebase -i <分支名稱>

繼續執行合併動作
Shell 
git rebase --continue

在解決衝突後，將剩餘的提交應用到目標分支上

恢復到合併前的狀態
Shell 
git rebase --abort

如果在合併過程中遇到問題或出現錯誤，可以使用此指令放棄重新應用

管理專案版本的常用指令

將文件加入索引中

新增了一個檔案A.txt，內容為aaa

Shell 
git add .

git add <文件名稱>（.代表目錄下的所有檔案）

查看索引的狀態
Shell 
git status

可以看到多了一個新檔案A.txt

將索引內的檔案提交到本地數據庫
Shell 
git commit -m "2023/08/04"

git commit -m <提交訊息>

將索引中的檔案提交到本地的數據庫中

查看提交的歷史訊息
Shell 
git log

查看剛剛commit的歷史訊息

將檔案推送到遠端數據庫
Shell 
git push origin master

git push <遠端名稱> <分支名稱>

已成功將本地數據庫的檔案推送到遠端數據庫

從遠端數據庫拉取檔案
Shell 
git pull origin master

git pull <遠端名稱> <分支名稱>

已成功將遠端數據庫的檔案拉取到本地數據庫
